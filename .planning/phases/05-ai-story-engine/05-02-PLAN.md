    ```typescript
    import { useState, useEffect, useRef, useCallback } from 'react';
    import { streamBeatProse } from '../lib/claudeStream';
    import type { UseStreamingTypewriterParams, UseStreamingTypewriterReturn } from './useStreamingTypewriter';
    // (keep the type exports from the skeleton)
    ```

    Implementation logic:
    1. State: `displayedText` (string, ''), `isComplete` (boolean, false)
    2. Refs: `abortRef` (AbortController), `bufferRef` (string, ''), `isCancelledRef` (boolean), `indexRef` (number, 0)
    3. `completeInstantly` callback: `abortRef.current.abort()`, set `isCancelledRef.current = true`, set `displayedText` to `bufferRef.current`, set `isComplete` to true
    4. Main `useEffect` with deps `[params.beatId]`:
       - If `params.isStaticBeat`: set `displayedText` to `params.fallbackText`, set `isComplete` to true, return early
       - Reset state: `bufferRef.current = ''`, `indexRef.current = 0`, `isCancelledRef.current = false`, set `displayedText('')`, set `isComplete(false)`
       - Create new AbortController: `abortRef.current = new AbortController()`
       - Define async function `runStream()`:
         - `try { for await (const chunk of streamBeatProse({ beatId: params.beatId, choiceHistory: params.choiceHistory, currentBeat: params.currentBeat, signal: abortRef.current.signal })) { if (isCancelledRef.current) return; bufferRef.current += chunk; setDisplayedText(bufferRef.current); } } catch (err) { if (err instanceof DOMException && err.name === 'AbortError') return; bufferRef.current = params.fallbackText; setDisplayedText(params.fallbackText); }`
         - After loop (still inside try, after for-await): if `!isCancelledRef.current`, set `isComplete(true)`
       - Call `runStream()`
       - Cleanup return: `() => { isCancelledRef.current = true; abortRef.current.abort(); }`
    5. Character-by-character reveal: Use chunk-level reveal for PoC (simpler, still progressive). Each yielded chunk (typically 1-5 words) is appended to displayedText immediately via `setDisplayedText(bufferRef.current)` inside the for-await loop. The streaming itself provides the progressive reveal effect as chunks arrive every ~50-100ms from the API.
    6. Return `{ displayedText, isComplete, completeInstantly }`

    The key insight: during streaming, `displayedText` updates on each chunk yield (progressive). After streaming completes or on error, the full text is set at once. `completeInstantly()` aborts the stream and sets the full buffer immediately.
