**tailwind.config.js** — Replace the generated file entirely:

```js
/** @type {import('tailwindcss').Config} */
export default {
  darkMode: 'selector',
  content: [
    './index.html',
    './src/**/*.{js,ts,jsx,tsx}',
  ],
  theme: {
    extend: {
      colors: {
        base: 'var(--color-base)',
        'rose-accent': 'var(--color-rose)',
        'purple-accent': 'var(--color-purple)',
        gold: 'var(--color-gold)',
        surface: 'var(--color-surface)',
        'text-primary': 'var(--color-text)',
        muted: 'var(--color-muted)',
      },
      fontFamily: {
        sans: ['"Space Grotesk Variable"', 'ui-sans-serif', 'system-ui', 'sans-serif'],
      },
    },
  },
  plugins: [],
};
```

Note: `darkMode: 'selector'` is correct for Tailwind v3.4+. Do NOT use `'class'`.

**src/index.css** — Replace the generated file entirely:

```css
@import '@fontsource-variable/space-grotesk';

@tailwind base;
@tailwind components;
@tailwind utilities;

:root {
  --color-base: #0D0B12;
  --color-rose: #D4799A;
  --color-purple: #9B7EC8;
  --color-gold: #D4AF37;
  --color-surface: #1A1724;
  --color-text: #F0EAF8;
  --color-muted: #8B7FA8;
}

.page-container {
  max-width: 1440px;
  margin: 0 auto;
  width: 100%;
}
```

The `@import` for the font MUST come before the `@tailwind` directives.

**index.html** — Add `class="dark"` to the `<html>` tag so the `darkMode: 'selector'` strategy activates immediately. The tag should read:
```html
<html lang="en" class="dark">
```

Also ensure the viewport meta tag is present in `<head>`:
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```
